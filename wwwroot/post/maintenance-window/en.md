# Maintenance Window After Cloud Cover

![Maintenance Window After Cloud Cover](https://commons.wikimedia.org/wiki/Special:FilePath/Clouds_over_the_Sun.jpg)

Cloud cover is a product feature if the interface knows what to do with it. We now use sustained low-contrast intervals to prompt calibration checks, cache cleanup, and panel temperature review.

The public screen says observing paused. The maintainer view says why: contrast below threshold for twelve minutes, image age stable, no stream error, roof sensor dry. That distinction matters. A visitor needs trust; a maintainer needs evidence.

The maintenance checklist is intentionally short: verify the last fresh frame, inspect the enclosure fan, run the focus motor through three positions, and confirm the display returns to quiet mode without a manual refresh.

A system that only works in perfect sunlight is not a solar product. It is a fair-weather demo.
