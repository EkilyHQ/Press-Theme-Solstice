# First Light Interface

![First Light Interface](https://commons.wikimedia.org/wiki/Special:FilePath/The_Sun_by_the_Atmospheric_Imaging_Assembly_of_NASA%27s_Solar_Dynamics_Observatory_-_20100819.jpg)

A good interface should make first light feel immediate. Not dramatic, not mystical, just available. The first screen in the classroom prototype now opens with the disk, the timestamp, and one status sentence: observing conditions are stable.

Solstice is the right theme for this demo because it handles large media, code blocks, quotes, and metadata with a polished product surface. The image gets room, and the controls do not become the subject.

The first release will ship with three modes: quiet, tracking, and alert. Quiet is for a wall display. Tracking is for instructors adjusting the telescope. Alert is for the rare moment when the system needs a human to stop trusting the image.

Open design question: should the timestamp read like an instrument value or a publication date? The current build uses both, but the next pass should make one of them primary.
